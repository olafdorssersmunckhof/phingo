# Phingo v2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild Phingo with host auth (invite-only), team-based play, manual refresh, and game management (edit + delete).

**Architecture:** iron-session encrypted-cookie auth protects all /host/* routes; bcryptjs hashes passwords. Players table is replaced by teams table with a unique 6-char join code per team; any team member can join on any device anonymously. Polling is removed in favour of a manual refresh button.

**Tech Stack:** Next.js 16 App Router, Neon (serverless PostgreSQL), UploadThing v7, iron-session v8, bcryptjs, Tailwind CSS v4
